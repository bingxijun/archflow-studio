# SketchUp bridge deployment

Use this reference when installing or repairing the ArchFlow-owned SketchUp connection on Windows.

## Boundary

The connection is entirely local:

```text
Codex Plugin MCP server -> 127.0.0.1:9877 -> ArchFlow::Bridge -> SketchUp Ruby API
```

It does not clone a Git repository, install a third-party MCP package, edit `.codex/config.toml`, register an account, or send telemetry. The Codex Plugin declares `archflow-sketchup` through its own `.mcp.json`. A random per-user token in `%LOCALAPPDATA%\ArchFlow\bridge-token` authenticates the local socket.

## Required writes

Applying setup writes only:

- `archflow_bridge.rb` and `archflow_bridge/` under the selected `%APPDATA%\SketchUp\SketchUp <version>\SketchUp\Plugins` directory;
- timestamped backups of earlier ArchFlow Bridge files;
- `%LOCALAPPDATA%\ArchFlow\bridge-token` when it does not already contain a valid token.

Do not delete or replace unrelated extensions. Do not automatically remove a separately installed legacy bridge; report it and let the user decide after confirming its origin.

## Procedure

Run the read-only plan first:

```powershell
powershell -ExecutionPolicy Bypass -File scripts\preflight_check.ps1
powershell -ExecutionPolicy Bypass -File scripts\deploy_sketchup_mcp.ps1 -PlanOnly
```

When the user explicitly asked to configure SketchUp and the detected destination is correct:

```powershell
powershell -ExecutionPolicy Bypass -File scripts\deploy_sketchup_mcp.ps1
```

The compatibility filename `deploy_sketchup_mcp.ps1` now deploys only the ArchFlow-owned bridge.

## Package contract

`assets/plugins/archflow_bridge.rbz` is built reproducibly from `assets/sketchup-extension/` with:

```powershell
python scripts\build_sketchup_rbz.py
```

The builder uses fixed ZIP metadata and stores the two small Ruby entries without compression. This avoids zlib-dependent archive hashes across supported Python versions and operating systems.

The RBZ must contain exactly:

```text
archflow_bridge.rb
archflow_bridge/main.rb
```

The hash is recorded in `assets/integration-lock.json`.

## Validation

Restart SketchUp and Codex, open a blank model, then check:

```powershell
Test-NetConnection 127.0.0.1 -Port 9877
```

Call `sketchup_bridge_status`, followed by `sketchup_get_scene_info` or `sketchup_get_selection`. These are read-only. Mutating tools must follow an explicit user request and should be verified by reading the scene again.

`sketchup_run_archflow_script` accepts only a `.rb` file whose first line identifies it as generated by ArchFlow, whose SHA-256 matches the supplied digest, and whose call sets `confirmed: true`. Inspect the file and obtain user approval before calling it.
